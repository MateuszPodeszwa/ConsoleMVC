# Goal & Purpose

This is an attempt to make console application to behave sa ASPNET CORE MVC Controller-ViewModel Pattern.
There are many console applications, and those are not very well designed, as the interface is never at the utmost importance.
Developers use many frameworks such as Spectre Consoel etc.. but these play a wider part in the overal code structure. 
As, if the code is a mess, no matter how well your UI looks like, it is a pain to maintain it. Therefore, I decided to create 
something tested, something we all know and inspire myself from the better pattern, the MVC. 
It is not full recreation of MVC pattern, as ASPNET CORE is massive, this is meant to be just a way to structure your console application code,
so it is ubelievably well defined, structure, maintainable and easy to follow and expand. Each page has its model, controller just as in regular MVC pattern.
